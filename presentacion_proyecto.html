El Servidor de Red

Librerías: fastapi, websockets

  • FastAPI / Sockets: Servidor para endpoints y sockets.
  • WebSockets: Conexión abierta para video en tiempo real.
¿Qué es Base64?

Codificación que convierte fotos JPEG binarias a texto. Permite enviar el video dentro de mensajes JSON normales por WebSocket.

Código en servidor_ia.py

Codificación a texto y envío de fotograma en tiempo real:

b64_str = base64.b64encode(jpeg).decode("ascii")
payload = {
    "type": "fotograma",
    "data": b64_str
}
await cliente.send_text(json.dumps(payload))

Auto-Descubrimiento

Librería: zeroconf (mDNS)

  • mDNS: Resuelve nombres de host de forma local.
  • Auto-conexión: El servidor se anuncia y la app Flutter se conecta sola sin configurar IPs.
¿Por qué lo usamos?

Evita ingresar la IP local del servidor a mano. Las IPs cambian cada vez que se reinicia el router local.

Anuncio en Código

Registro del servicio local al inicializar la red:

info = ServiceInfo(
    "_alertavecinal._tcp.local.",
    "ServidorIA._alertavecinal...",
    addresses=[ip_local],
    port=8765
)
zeroconf.register_service(info)

Lectura de Matrículas

Librería: easyocr

  • EasyOCR: Traduce caracteres visuales a texto digital.
  • Allowlist: Limita a letras A-Z y números, ignorando tornillos o marcas de placa.
¿Por qué usar subprocesos (ThreadPool)?

El OCR tarda ~100ms. Ejecutarlo en segundo plano mantiene la transmisión de video fluida.

Lógica del OCR Asíncrono

Análisis en segundo plano de forma no bloqueante:

# Enviar tarea OCR a pool de hilos
ejecutor = ThreadPoolExecutor(max_workers=2)
ejecutor.submit(_ocr_task, id_r, img_placa)

Preprocesamiento de Imagen

Librería: opencv-python

OpenCV optimiza la placa para que el OCR no falle:

  • Escalado (`cv2.resize`): Forzamos altura de 100 píxeles para estandarizar caracteres.
  • Grayscale (`cv2.cvtColor`): Descarta colores para reducir datos y ahorrar procesamiento.

Implementación OpenCV

Normalización de tamaño y canales de color:

scale = 100.0 / plate_height
resized = cv2.resize(roi_placa, None, 
    fx=scale, fy=scale, 
    interpolation=cv2.INTER_LANCZOS4)

gray = cv2.cvtColor(resized, cv2.COLOR_BGR2GRAY)

Filtros Avanzados OpenCV

CLAHE & Umbral Adaptativo

  • CLAHE: Ecualiza contraste por zonas para resaltar letras bajo brillos.
  • Umbral Adaptativo: Convierte a blanco y negro absoluto adaptando la luz de cada zona.
¿Por qué Umbral Adaptativo?

Un umbral fijo falla ante sombras. El adaptativo calcula cortes de binarización por píxel según sus vecinos.

Lógica OpenCV

Ecualización y binarización inversa:

clahe = cv2.createCLAHE(clipLimit=2.0)
contrast = clahe.apply(gray_plate)

thresh = cv2.adaptiveThreshold(contrast, 255,
    cv2.ADAPTIVE_THRESH_GAUSSIAN_C,
    cv2.THRESH_BINARY_INV, 11, 2)

Base de Datos y Búsqueda Difusa

Librerías: sqlite3, difflib

  • SQLite (`database.py`): Almacén de placas reportadas e historial local de alertas.
  • difflib: Compara el parecido de textos.
¿Por qué usar Búsqueda Difusa?

Si el OCR lee mal un carácter (ej. 'B' por '8'), detecta similitud > 80% y dispara la alerta vecinal.

Código de Comparación Difusa

Cálculo de similitud aproximada contra la BD:

sim = difflib.SequenceMatcher(None, 
    texto_detectado, placa_bd).ratio()

if sim >= 0.80:
    registrar_alerta_robo()
